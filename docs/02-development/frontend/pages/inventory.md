# صفحة إدارة المخزون (Inventory)

## نظرة عامة

صفحة إدارة المخزون تسمح للمستخدمين بمراقبة وإدارة مخزونهم: عرض مستويات المخزون، حركات المخزون، التنبيهات، والتحويلات.

**الموقع**: `src/pages/Inventory/Inventory.tsx`  
**المسار**: `/inventory`  
**الحالة**: ✅ مكتملة

---

## الوظائف الرئيسية

### 1. عرض المخزون
- جدول عناصر المخزون
- عرض الكمية لكل منتج في كل مخزن
- تنبيهات النقص
- إحصائيات المخزون

### 2. إدارة المخزون
- تعديل كمية المخزون
- تحويل بين المخازن
- عرض حركات المخزون
- تسوية المخزون

### 3. التنبيهات
- تنبيهات المخزون المنخفض
- تنبيهات نفاد المخزون
- تنبيهات نقطة إعادة الطلب

### 4. التقارير
- تقرير المخزون الشامل
- تقرير الحركات
- تقرير المخزون المنخفض

---

## المكونات المستخدمة

- `InventoryTable`: جدول المخزون المتخصص
- `StockItemForm`: نموذج تعديل المخزون
- `WarehouseTransferForm`: نموذج التحويل
- `KPICard`: بطاقات الإحصائيات

---

## تدفق المستخدم (User Flow)

### تدفق تعديل المخزون
1. المستخدم ينقر على "تعديل" في جدول المخزون
2. يفتح نموذج تعديل المخزون
3. يعدل الكمية
4. يضيف سبب التعديل
5. يحفظ

### تدفق تحويل المخزون
1. المستخدم ينقر على "تحويل"
2. يختار المخزن المصدر والهدف
3. يحدد الكمية
4. يحفظ

---

## APIs المستخدمة

- `GET /inventory/stock-items` - جلب عناصر المخزون
- `PATCH /inventory/stock-items/:id` - تحديث المخزون
- `POST /inventory/transfers` - تحويل المخزون
- `GET /inventory/movements` - جلب الحركات
- `GET /inventory/alerts` - جلب التنبيهات

---

## حالات الاستخدام

### حالة الاستخدام 1: مراقبة المخزون
المستخدم يريد رؤية حالة المخزون:
1. يفتح صفحة Inventory
2. يرى جدول المخزون
3. يرى التنبيهات للمخزون المنخفض
4. يمكنه فلترة حسب المخزن

### حالة الاستخدام 2: تعديل المخزون
المستخدم يريد تعديل كمية منتج:
1. يبحث عن المنتج
2. ينقر "تعديل"
3. يعدل الكمية
4. يحفظ
